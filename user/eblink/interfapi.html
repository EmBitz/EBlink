4.8. InterfAPI

InterfAPI is a class registered by EBlink that represents the physical debug probe (ST-Link, CMSIS-DAP, etc.). Scripts instantiate it once at global scope and share the instance across all included files:

intrfApi <- InterfAPI()

All methods that communicate with hardware throw an integer ERROR_xxx exception on failure.

4.8.1. Probe information

InterfAPI.getName()

Returns the interface name string (e.g. "stlink").

InterfAPI.getTapId()

Returns the JTAG TAP ID / SWD target ID as a 32-bit integer. Returns 0 if the read fails.

InterfAPI.getSpeed()

Returns the current probe communication speed in KHz.

InterfAPI.setSpeed(speed)

Sets the probe speed to speed KHz. Throws on error:

intrfApi.setSpeed(4000);
InterfAPI.targetVoltage()

Returns the measured target VCC voltage as a float (volts).

InterfAPI.resetCtrl(state)

Asserts (state = true) or deasserts (state = false) the hardware nRESET line. Throws on error.

4.8.2. DAP access

These functions provide direct access to the ARM Debug Access Port (DAP) registers, bypassing the normal memory interface. Useful for reading the IDCODE register or configuring AP/DP registers during device identification.

InterfAPI.setDapAp(ap)

Selects the active Access Port (0–127). Returns the previously selected AP number. Throws on error.

InterfAPI.getDapAp()

Returns the currently selected Access Port number.

InterfAPI.readDap(ap, addr)

Reads a 32-bit value from DAP register addr on access port ap. Throws on error.

InterfAPI.writeDap(ap, addr, value)

Writes a 32-bit value to DAP register addr on access port ap. Throws on error.

4.8.3. Memory read

InterfAPI.read(addr[, width])

Reads a single value from target address addr. width is 8, 16, or 32 bits. When omitted (or -1), uses the probe’s native 32-bit debug access path. Returns the value as an integer. Throws on error:

local cpuid = intrfApi.read(0xE000ED00);
printf("CPUID: 0x%08X\n", cpuid);
InterfAPI.readBuf(buffer, addr, length[, width])

Reads length bytes from target address addr into buffer (an EBbuffer()). The buffer is (re-)allocated to length bytes internally — no prior allocation is needed. width controls bus transfer width (8, 16, or 32); omit or pass -1 for the default. Returns length on success. Throws on error:

local buf = EBbuffer();
intrfApi.readBuf(buf, 0x08000000, 1024);

4.8.4. Memory write

InterfAPI.write(addr, value[, width])

Writes a single value to target address addr. width is 8, 16, or 32 bits; defaults to 32-bit debug access when omitted. Throws on error.

InterfAPI.writeBuf(buffer, addr[, width[, offset, length]])

Writes the contents of buffer to target address addr.

Optional parameters:

  • width — bus transfer width: 8, 16, or 32. Omit or -1 for default.

  • offset — byte offset into buffer to start from (must be paired with length).

  • length — number of bytes to write (required when offset is given).

Throws on error:

// Write entire buffer
intrfApi.writeBuf(buf, 0x08000000);

// Write 256 bytes starting at offset 512
intrfApi.writeBuf(buf, 0x08001000, -1, 512, 256);

4.8.5. File loading

InterfAPI.loadBin(filename, addr)

Reads a binary file from disk and writes it to target memory starting at addr. EBlink searches the configured script paths for the file. Returns the number of bytes written. Throws an integer exception (via ERROR_NOTIFIED) if the file cannot be opened.

InterfAPI.loadString(addr, str, len)

Writes the raw bytes of str (up to len bytes) to target address addr. Returns len. Throws on hardware error. Useful for writing null-terminated strings or packed data to target RAM:

intrfApi.loadString(0x20000000, "BOOT", 4);
