4.10. TargetAPI

TargetAPI is a class registered by EBlink that controls the CPU state: halting, resuming, reading and writing registers, executing code, and resetting the device. Scripts instantiate it once at global scope:

targetApi <- TargetAPI()

All methods throw an integer ERROR_xxx exception on failure.

4.10.1. Target information

TargetAPI.getName()

Returns the device name string as reported by the target driver (e.g. "STM32F4") or null if no target is attached.

TargetAPI.getCpuName()

Returns the ARM CPU core name (e.g. "M0", "M3", "M4").

TargetAPI.getState()

Returns the current CPU state as one of the TARGET_xxx constants: TARGET_UNKNOWN, TARGET_RUNNING, TARGET_HALTED, TARGET_RESET.

4.10.2. Execution control

TargetAPI.halt()

Requests the CPU to halt. Throws on error.

TargetAPI.resume()

Resumes a halted CPU. Throws on error.

TargetAPI.poll()

Polls the target hardware and updates the internal state. Call this after resume() before reading getState() if you need an immediate state update.

TargetAPI.execute(addr, wait)

Writes addr to the program counter (PC register) and resumes the CPU. If wait is true, blocks until the CPU halts again. Useful for running a small RAM-resident flash algorithm:

// Upload algorithm to RAM, then execute it
intrfApi.writeBuf(algo_buf, 0x20000000);
targetApi.execute(0x20000000, true);
TargetAPI.reset(type)

Resets the device using one of the xxx_RESET constants:

targetApi.reset(SYSTEM_RESET);

For USER_RESET, the ebResetUser() script function is called instead of a hardware reset. After the reset the ebResetPost() function is called if it exists.

4.10.3. Register access

Register names are core-specific. For ARM Cortex-M the standard names are "r0" through "r15", "pc" (same as "r15"), "sp" (same as "r13"), "lr" (same as "r14"), and "xpsr".

TargetAPI.readReg(name)

Returns the 32-bit value of the named register. Throws on error or if the register name is unknown:

local pc = targetApi.readReg("pc");
printf("PC = 0x%08X\n", pc);
TargetAPI.writeReg(name, value)

Writes value to the named register. Throws on error or if the register name is unknown:

targetApi.writeReg("r0", 0x00000000);  // clear R0

4.10.4. Example: read and print all core registers

function dump_regs()
{
    local regs = ["r0","r1","r2","r3","r4","r5","r6","r7",
                  "r8","r9","r10","r11","r12","sp","lr","pc","xpsr"];
    foreach(name in regs)
        printf("%-6s = 0x%08X\n", name, targetApi.readReg(name));
}
