4.5. Constants

All constants listed here are available in every EBlink script without declaration.

4.5.1. VERSION

VERSION is a float containing the EBlink API version:

if(VERSION < 3.0)
    errorf("This script requires EBlink API 3.0 or newer\n");

4.5.2. Target state

Returned by TargetAPI.getState().

Constant

Value

Meaning

TARGET_UNKNOWN

0

State cannot be determined

TARGET_RUNNING

1

CPU is executing code

TARGET_HALTED

2

CPU is halted (debug state)

TARGET_RESET

3

CPU is held in reset

4.5.3. Reset types

Passed to TargetAPI.reset().

Constant

Value

Effect

SYSTEM_RESET

0

Full system reset via AIRCR.SYSRESETREQ

CORE_RESET

1

Core-only reset via AIRCR.VECTRESET

JTAG_RESET

2

Hardware reset via the nRESET pin

USER_RESET

3

Custom reset — calls ebResetUser()

4.5.4. Error codes

All negative integers thrown by API methods correspond to one of these constants. Scripts can compare a caught integer exception against them:

catch(e) {
    if(e == ERROR_TARGET_LOST) { errorf("Target disconnected\n"); return e; }
    throw e;
}

User-defined error codes must be more negative than ERROR_USER:

const MY_UNLOCK_FAIL = (ERROR_USER - 1);  // -1001

INTERFACE_WAIT (-21) usually means the probe speed set with iface.setSpeed() is too high for the target. Try reducing it or use the -D FLASH_SPEED=<KHz> CLI option.
